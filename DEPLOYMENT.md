# Deployment Guide - ELD Trip Planner

## Overview

This guide covers deploying the ELD Trip Planner application to production using Vercel (frontend) and Railway/Render (backend).

## Prerequisites

- GitHub account (for deploying from repo)
- Vercel account (free tier available)
- Railway or Render account (for backend)
- Custom domain (optional)

## Step 1: Prepare Application for Production

### Backend Changes

1. **Update settings.py**
```python
DEBUG = False
ALLOWED_HOSTS = ['your-backend-domain.com']
SECRET_KEY = os.environ.get('SECRET_KEY')

# Add production database
if os.environ.get('DATABASE_URL'):
    import dj_database_url
    DATABASES = {
        'default': dj_database_url.config(default=os.environ.get('DATABASE_URL'))
    }
```

2. **Add production requirements** (in requirements.txt):
```
gunicorn==21.0.0
dj-database-url==2.0.0
whitenoise==6.5.0
psycopg2-binary==2.9.6  # For PostgreSQL
```

3. **Update CORS settings**:
```python
CORS_ALLOWED_ORIGINS = [
    "https://your-frontend-domain.com",
    "https://your-backend-domain.com",
]
```

4. **Add WhiteNoise middleware** (for static files):
```python
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'whitenoise.middleware.WhiteNoiseMiddleware',  # Add this
    # ... rest of middleware
]
```

### Frontend Changes

1. **Update API endpoint** in `TripForm.js`:
```javascript
const API_URL = process.env.REACT_APP_API_URL || 'http://localhost:8000/api';
```

2. **Create .env.production**:
```
REACT_APP_API_URL=https://your-backend-domain.com/api
```

3. **Update package.json** homepage:
```json
"homepage": "https://your-frontend-domain.com"
```

## Step 2: Deploy Backend to Railway

### Option A: Railway.app (Recommended)

1. **Create Railway Account**
   - Go to railway.app
   - Sign up with GitHub

2. **Create New Project**
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Connect your GitHub account
   - Select the repository

3. **Configure Environment**
   - In Railway dashboard, go to "Variables"
   - Add environment variables:
     ```
     DEBUG=False
     SECRET_KEY=[Generate a secure key]
     ALLOWED_HOSTS=your-railway-domain.com
     DATABASE_URL=[Auto-generated by Railway]
     CORS_ALLOWED_ORIGINS=https://your-vercel-domain.vercel.app
     ```

4. **Create Procfile** in backend directory:
```
web: gunicorn config.wsgi
release: python manage.py migrate
```

5. **Add Railway configuration** (railway.toml):
```toml
[build]
builder = "nixpacks"

[deploy]
restartPolicyType = "on_failure"
restartPolicyMaxRetries = 2
```

6. **Deploy**
   - Push to GitHub
   - Railway auto-deploys

### Option B: Heroku

1. **Install Heroku CLI**
2. **Create Procfile**:
```
web: gunicorn config.wsgi
release: python manage.py migrate
```

3. **Deploy**:
```bash
heroku login
heroku create your-app-name
heroku config:set SECRET_KEY="your-secret-key"
git push heroku main
```

## Step 3: Deploy Frontend to Vercel

### Deploy via GitHub

1. **Create Vercel Account**
   - Go to vercel.com
   - Sign up with GitHub

2. **Import Project**
   - Click "New Project"
   - Select your repository
   - Select "eld-app/frontend" as root directory

3. **Configure Environment Variables**
   - Add `REACT_APP_API_URL` = your backend URL
   - Add `REACT_APP_API_URL` for preview deployments

4. **Build Settings**
   - Build Command: `npm run build`
   - Output Directory: `build`
   - Install Command: `npm install`

5. **Deploy**
   - Click "Deploy"
   - Vercel automatically deploys on push to main branch

### Manual Deployment

```bash
# Install Vercel CLI
npm install -g vercel

# In frontend directory
cd eld-app/frontend
vercel --prod
```

## Step 6: Test Production Deployment

1. **Test Backend API**
```bash
curl -X GET https://your-backend-domain.com/api/trips/
```

2. **Test Frontend**
   - Visit https://your-frontend-domain.com
   - Enter test trip data
   - Verify map and logs are generated

3. **Test CORS**
   - Open browser DevTools
   - Check for CORS errors

## Step 7: Setup CI/CD Pipeline

### GitHub Actions (Optional)

Create `.github/workflows/deploy.yml`:
```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  backend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-python@v2
        with:
          python-version: 3.9
      - run: pip install -r eld-app/backend/requirements.txt
      - run: cd eld-app/backend && python manage.py test

  frontend-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-node@v2
        with:
          node-version: 16
      - run: npm install --prefix eld-app/frontend
      - run: npm run build --prefix eld-app/frontend
```

## Troubleshooting

### CORS Errors
```
Error: Access to XMLHttpRequest blocked by CORS policy
```

**Solution:**
- Update `CORS_ALLOWED_ORIGINS` in backend settings
- Ensure API URL in frontend matches deployment domain
- Verify backend is accessible

### Static Files Not Loading
```
Error: 404 not found for static files
```

**Solution:**
```python
# In settings.py
STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
```

### Database Migrations
```
django.db.utils.OperationalError: no such table
```

**Solution:**
- Ensure migrations run on deployment
- Check Procfile has `release: python manage.py migrate`

### Environment Variables Not Loading
```
KeyError: 'SECRET_KEY'
```

**Solution:**
- Check all environment variables are set in Railway/Vercel
- Verify variable names match exactly (case-sensitive)

## Scaling Considerations

### When to Scale
- 1,000+ requests/day → Upgrade database
- 10,000+ requests/day → Add caching layer (Redis)
- 100,000+ requests/day → Consider CDN + multiple regions

### Scaling Options
- **Horizontal**: Add more servers/containers
- **Vertical**: Increase server resources
- **Database**: Upgrade to managed PostgreSQL
- **CDN**: Use Cloudflare or similar
